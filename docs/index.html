Artifact + Registry

One SQLite datastore where plugins can coexist without collisions.

Artistry gives every artifact kind a registry-backed identity, stores flexible JSON payloads with schema validation, and keeps immutable version history by default through copy-on-write updates.

  • Global IDs across all kinds, addressable with short slugs like E42.
  • JSON Schema subset support with defaults, constraints, and optional fields.
  • Typed links, normalized tags, and transaction helpers for atomic workflows.

Core Concepts

Built for collaborative data systems

Registry

Register kinds with plugin name, schema, and a compact assigned code.

Unified Identity

Auto-increment IDs are global across all artifacts, not siloed per kind.

Versioning

Default updates create a new current version while preserving lineage.

Validation

JSON schema constraints enforce data shape, defaults, and required keys.

Links

Create typed cross-artifact relations with cascade cleanup semantics.

Tags

Normalized tags support OR and AND lookup patterns across artifacts.

Quick Start

Example registration

require "artistry"

Artistry.open("myapp.db")

Artistry::Registry.register(
  kind: "event",
  plugin: "myapp",
  schema: {
    type: "object",
    properties: {
      title: {type: "string", minLength: 1},
      date: {type: "string", format: "date"},
    },
    required: ["title", "date"],
    additionalProperties: false,
  }
)

event = Artistry::Artifact.create("event", {title: "Team Meeting", date: "2025-06-15"})
Artistry::Tag.tag(event, "mvp")
